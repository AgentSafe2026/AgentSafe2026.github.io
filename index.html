Fully Autonomous Agent Safety: Evaluation, Control, and Assurance

About the Workshop

Agent safety has long been studied under an assistant model of agent use: a user issues a request, the system proposes an action, the user reviews and approves each step. Recent systems are moving toward fully autonomous operation — production coding agents, self-hosted long-running agents, and autonomous research systems where the user states a high-level objective and steps away. This workshop focuses on how to make unattended agent runs safe, measurable, and trustworthy. We center on three questions: how should we evaluate the final result, the agent's own success claim, and the side effects produced during execution? How should we define and test autonomy boundaries? What evidence should accompany an unattended run before developers, organizations, or regulators can trust it? We welcome contributions from foundation model agents, software engineering agents, autonomous research systems, AI security and red teaming, scalable oversight, sandboxing, human-AI interaction, evaluation, and governance.

Topics of Interest

We invite submissions on the following topics and beyond.

Evaluation & Benchmarking

  • Goal-to-deliverable benchmarks and metrics
  • Self-verification and false completion detection
  • Measuring errors in long-horizon tasks
  • Distinguishing genuine completion from overconfident claims
  • Prompt injection and tool misuse benchmarks

Control & Authority Design

  • Autonomy levels and action budgets
  • Approval checkpoints and tool allowlists
  • Reversibility constraints and runtime monitors
  • Credential and secret protection
  • Secure sandboxed execution environments

Adversarial Robustness

  • Unattended agent red teaming
  • Prompt injection and tool poisoning
  • Sandbox escape and privilege escalation
  • Data exfiltration and adaptive attacks
  • Defenses for high-privilege autonomous agents

Long-horizon & Multi-agent Safety

  • Persistent memory and scheduled task safety
  • Compounding errors and delayed failure recovery
  • Subagent spawning and delegation
  • Assurance, auditability, and trace-level telemetry
  • Governance, safety cases, and regulatory interfaces
